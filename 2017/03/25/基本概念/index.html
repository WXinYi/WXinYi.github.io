（一）一些基本概念

发表于 2017-03-25 | 分类于 技术   |    阅读次数

通过查看objc.h 了解一些基本关键词的底层定义

SEL/objc_selector:

objc_selector 

透明的数据结构(C string)

SEL 

一个指向C String(objc_selector)的指针
 代码定义：
     typedef struct objc_selector *SEL;

id/objc_object:

id 

指向一个类的实例对象 
代码定义：
    tyoedef struct objc_object *id;

objc_object： 

保存了一个Class 类型的isa
代码定义：
    struct objc_object{
        class isa OBJC_ISA_AVAILABILITY;  
    }

Class/objc_class

Class

指向Objective-c对象的一个指针
代码定义：
  typedef struct objc_class *Class;

objc_class

代码定义：
  struct objc_class {
      Class isa  OBJC_ISA_AVAILABILITY;

  #if !__OBJC2__
      Class super_class                       OBJC2_UNAVAILABLE;  // 父类
      const char *name                        OBJC2_UNAVAILABLE;  // 类名
      long version                            OBJC2_UNAVAILABLE;  // 类的版本信息，默认为0
      long info                               OBJC2_UNAVAILABLE;  // 类信息，供运行期使用的一些位标识
      long instance_size                      OBJC2_UNAVAILABLE;  // 该类的实例变量大小
      struct objc_ivar_list *ivars            OBJC2_UNAVAILABLE;  // 该类的成员变量链表
      struct objc_method_list **methodLists   OBJC2_UNAVAILABLE;  // 方法定义的链表
      struct objc_cache *cache                OBJC2_UNAVAILABLE;  // 方法缓存
      struct objc_protocol_list *protocols    OBJC2_UNAVAILABLE;  // 协议链表
  #endif

  } OBJC2_UNAVAILABLE;

IMP

IMP - 指向实际执行函数体的函数指针
代码定义：
    #if !OBJC_OLD_DISPATCH_PROTOTYPES
    typedef void (*IMP)(void /* id, SEL, ... */ ); 
    #else
    typedef id (*IMP)(id, SEL, ...); 
    #endif

需要至少两个参数id（消息的接受者，也就是对象），SEL(方法的名字)，后面是方法的实际参数列表

method/objc_method

method - 指向Objective C中的方法指针
代码定义：
   typedef struct objc_method *Method;

objc_method：
    struct objc_method {
    SEL method_name             OBJC2_UNAVAILABLE; // 方法名
    char *method_types          OBJC2_UNAVAILABLE;
    IMP method_imp              OBJC2_UNAVAILABLE; // 方法实现
   }
 这个结构体中包含一个SEL和IMP，相当于在SEL和IMP之间作了一个映射。
 有了SEL，就可以找到对应的IMP，从而调用方法的实现代码。
当我们向一个对象发送消息时，runtime会在这个对象所属的这个类的方法列表中查找方法；而向一个类发送消息时，会在这个类的meta-class的方法列表中查找。
